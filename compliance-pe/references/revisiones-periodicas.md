# Revisiones periódicas

Recomendación mínima:

- Re-ejecutar `/compliance-pe` cada trimestre.
- Re-ejecutar antes de lanzar funciones que recopilen nuevos datos.
- Re-ejecutar al contratar proveedores nuevos.
- Re-ejecutar después de incidentes de seguridad.
- Revisar anualmente política de privacidad, documento de seguridad, banco de datos y contratos de encargo.

Sugerencia de commit:

```bash
git add .compliance
git commit -m "chore: update data protection compliance review"
```
