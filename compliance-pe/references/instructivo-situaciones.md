# Instructivo de situaciones frecuentes

## 1. Solicitud ARCO

1. Registrar fecha y canal de ingreso.
2. Verificar identidad del solicitante sin recopilar datos excesivos.
3. Clasificar: acceso, actualización, inclusión, rectificación, supresión, impedir suministro, oposición o tratamiento objetivo.
4. Ubicar sistemas y bancos de datos afectados.
5. Bloquear datos cuando corresponda durante actualización/supresión.
6. Responder con lenguaje claro y guardar evidencia.
7. Si se deniega, fundamentar y dejar constancia del mecanismo de tutela ante ANPD o vía judicial.

## 2. Incidente de seguridad

1. Abrir incidente con hora exacta de conocimiento.
2. Identificar naturaleza, datos, titulares aproximados, sistemas y proveedor involucrado.
3. Contener: revocar credenciales, aislar sistemas, rotar secretos, bloquear accesos.
4. Evaluar si hay grandes volúmenes, datos sensibles, muchos titulares o perjuicio a derechos.
5. Si corresponde, notificar a ANPD dentro de 48 horas desde conocimiento.
6. Si afecta derechos del titular, comunicar al titular dentro de 48 horas, sin dilación indebida.
7. Si ocurre en entorno digital, considerar notificación adicional al Centro Nacional de Seguridad Digital.
8. Documentar hechos, efectos y medidas adoptadas.

## 3. Fiscalización ANPD

1. Centralizar comunicación en representante legal, Oficial de Datos Personales o responsable interno.
2. Preservar evidencia: logs, contratos, políticas, consentimientos, registro de bancos, incidentes.
3. No alterar ni borrar registros.
4. Responder dentro de plazos formales.
5. Acudir a abogado si existe procedimiento sancionador, medida correctiva, denuncia o requerimiento complejo.

## 4. Nuevo proveedor

1. Identificar datos tratados y país.
2. Determinar rol: encargado, subencargado, tercero receptor.
3. Revisar seguridad, subprocesadores, ubicación y certificaciones.
4. Firmar contrato de encargo o cláusulas equivalentes.
5. Actualizar política, inventario, banco de datos y transferencia internacional si corresponde.

## 5. Nueva funcionalidad de IA, analytics o perfilamiento

1. Determinar si se usan datos personales o anonimizados.
2. Evitar enviar datos sensibles o innecesarios a proveedores externos.
3. Evaluar EIPD si hay perfilamiento, decisiones automatizadas, grandes volúmenes o datos sensibles.
4. Informar al titular de finalidades y terceros.
5. Incorporar revisión humana si puede afectar significativamente al titular.
